# AIShooting 단계별 수행 절차

## 목적

`Assets/Tasks`의 구현 작업을 순서대로 진행한 뒤, Unity Editor에서 프로토타입 핵심 루프를 실습하고 검증하기 위한 단계별 절차를 정리한다.

## 공통 전제

1. Unity Editor 버전은 `ProjectSettings/ProjectVersion.txt` 기준 `6000.4.10f1`을 사용한다.
2. Game View 기준 해상도는 세로형 `768 x 1024`로 설정한다.
3. 주요 게임플레이 카메라는 `Projection = Orthographic`으로 설정한다.
4. 플레이어, 적, 총알의 이동은 XY 평면 기준으로 구성하고, 위치는 `Vector3(x, y, 0)` 기준을 유지한다.
5. 게임플레이 입력은 Old Input Manager API 기준으로 검증한다.
6. 각 Task를 완료할 때마다 대응되는 `Assets/Checklists`의 체크리스트를 확인한다.

## 1. 프로젝트 기본 씬 준비

1. Unity Editor에서 프로젝트를 연다.
2. 테스트에 사용할 씬을 열거나 새 씬을 만든다.
3. `Main Camera`의 Projection을 `Orthographic`으로 변경한다.
4. 카메라가 XY 플레이 평면을 바라보도록 배치한다.
5. Game View 해상도를 `768 x 1024` 또는 동일한 세로 비율로 설정한다.
6. 씬의 주요 게임플레이 오브젝트는 `z = 0` 주변에 배치한다.

## 2. 스크립트 폴더 준비

1. `Assets/Scripts` 폴더가 없으면 만든다.
2. `Assets/Scripts/Gameplay` 폴더가 없으면 만든다.
3. `Assets/Scripts/Core` 폴더가 없으면 만든다.
4. `Assets/Scripts/UI` 폴더가 없으면 만든다.

## 3. `HO_PlayerController` 구현 및 플레이어 구성

1. `Assets/Tasks/01_HO_PlayerController.md` 기준으로 `Assets/Scripts/Gameplay/HO_PlayerController.cs`를 구현한다.
2. 씬에 플레이어용 게임오브젝트를 만든다.
   - 예: `Capsule`, `Cube`, 또는 임시 기체 모델.
3. 플레이어 게임오브젝트 이름을 `Player`로 변경한다.
4. `Player` 위치를 화면 하단 영역의 `Vector3(x, y, 0)` 위치로 배치한다.
5. `Player`에 `HO_PlayerController` 컴포넌트를 추가한다.
6. `Player`에 충돌 검증용 Collider를 추가한다.
7. 충돌 이벤트가 필요하면 Rigidbody를 추가하고, Z축 방향 이동이 게임플레이에 섞이지 않도록 설정한다.
8. `Player` 하위에 빈 게임오브젝트를 만들고 이름을 `FirePoint`로 변경한다.
9. `FirePoint`를 플레이어보다 위쪽에 배치한다.
10. `HO_PlayerController`의 Fire Point 필드에 `FirePoint`를 할당한다.
11. 이동 속도, 이동 최소 경계, 이동 최대 경계, 최대 HP 값을 Inspector에서 설정한다.
12. 총알 프리팹은 `HO_PlayerBullet` 프리팹을 만든 뒤 연결한다.
13. Play Mode에서 `Horizontal`, `Vertical` 입력으로 XY 평면 이동을 확인한다.
14. `Assets/Checklists/01_HO_PlayerController_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 4. `HO_PlayerBullet` 구현 및 총알 프리팹 구성

1. `Assets/Tasks/02_HO_PlayerBullet.md` 기준으로 `Assets/Scripts/Gameplay/HO_PlayerBullet.cs`를 구현한다.
2. 씬에 총알용 게임오브젝트를 만든다.
   - 예: 작은 `Sphere` 또는 `Capsule`.
3. 총알 게임오브젝트 이름을 `PlayerBullet`로 변경한다.
4. `PlayerBullet` 위치는 임시로 `Vector3(0, 0, 0)`에 둔다.
5. `PlayerBullet`에 `HO_PlayerBullet` 컴포넌트를 추가한다.
6. `PlayerBullet`에 충돌 검증용 Collider를 추가한다.
7. 충돌 이벤트가 필요하면 Rigidbody를 추가하고, Z축 방향 이동이 게임플레이에 섞이지 않도록 설정한다.
8. `HO_PlayerBullet`의 이동 속도, 대미지, 화면 위쪽 제거 Y 값을 Inspector에서 설정한다.
9. `PlayerBullet`을 Project 창으로 드래그해 프리팹으로 만든다.
10. 씬에 남은 임시 `PlayerBullet` 오브젝트는 제거한다.
11. `Player`의 `HO_PlayerController` 컴포넌트에 총알 프리팹을 할당한다.
12. Play Mode에서 `Fire1` 입력 시 `FirePoint` 위치에서 총알이 생성되는지 확인한다.
13. 생성된 총알이 월드 좌표 기준 위쪽으로 이동하는지 확인한다.
14. `Assets/Checklists/02_HO_PlayerBullet_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 5. `HO_Enemy` 구현 및 적 프리팹 구성

1. `Assets/Tasks/03_HO_Enemy.md` 기준으로 `Assets/Scripts/Gameplay/HO_Enemy.cs`를 구현한다.
2. 씬에 적 게임오브젝트를 만든다.
   - 예: 플레이어와 구분되는 색의 `Cube` 또는 `Capsule`.
3. 적 게임오브젝트 이름을 `Enemy`로 변경한다.
4. `Enemy` 위치는 임시로 화면 위쪽의 `Vector3(x, y, 0)` 위치에 둔다.
5. `Enemy`에 `HO_Enemy` 컴포넌트를 추가한다.
6. `Enemy`에 충돌 검증용 Collider를 추가한다.
7. 충돌 이벤트가 필요하면 Rigidbody를 추가하고, Z축 방향 이동이 게임플레이에 섞이지 않도록 설정한다.
8. `HO_Enemy`의 이동 속도, 접촉 대미지, 화면 아래쪽 제거 Y 값을 Inspector에서 설정한다.
9. `Enemy`를 Project 창으로 드래그해 프리팹으로 만든다.
10. 씬에 남은 임시 `Enemy` 오브젝트는 필요에 따라 제거하거나 충돌 테스트용으로 유지한다.
11. Play Mode에서 적이 월드 좌표 기준 아래쪽으로 이동하는지 확인한다.
12. 총알과 적이 충돌하면 적과 총알이 제거되는지 확인한다.
13. 적과 플레이어가 충돌하면 플레이어 HP가 감소하고 적이 제거되는지 확인한다.
14. `Assets/Checklists/03_HO_Enemy_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 6. `HO_GameManager` 구현 및 게임 상태 구성

1. `Assets/Tasks/04_HO_GameManager.md` 기준으로 `Assets/Scripts/Core/HO_GameManager.cs`를 구현한다.
2. 씬에 빈 게임오브젝트를 만들고 이름을 `GameManager`로 변경한다.
3. `GameManager`에 `HO_GameManager` 컴포넌트를 추가한다.
4. 적 처치 점수와 시작 시 게임 진행 여부를 Inspector에서 설정한다.
5. UI 컨트롤러는 `HO_UIController`를 만든 뒤 연결한다.
6. 총알로 적을 제거했을 때 점수가 증가하도록 `HO_Enemy`, `HO_PlayerBullet`, `HO_GameManager`의 연결 흐름을 확인한다.
7. 플레이어 사망 시 게임 종료 상태로 전환되는지 확인한다.
8. 게임 종료 상태를 다른 시스템에서 조회할 수 있는지 확인한다.
9. `Assets/Checklists/04_HO_GameManager_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 7. `HO_EnemySpawner` 구현 및 적 스폰 구성

1. `Assets/Tasks/05_HO_EnemySpawner.md` 기준으로 `Assets/Scripts/Gameplay/HO_EnemySpawner.cs`를 구현한다.
2. 씬에 빈 게임오브젝트를 만들고 이름을 `EnemySpawner`로 변경한다.
3. `EnemySpawner` 위치를 화면 위쪽 경계 근처 또는 경계 밖에 둔다.
4. `EnemySpawner`에 `HO_EnemySpawner` 컴포넌트를 추가한다.
5. Enemy Prefab 필드에 `Enemy` 프리팹을 할당한다.
6. Game Manager 필드에 씬의 `GameManager`를 할당한다.
7. 스폰 간격, X 최소값, X 최대값, Y 스폰 위치를 Inspector에서 설정한다.
8. 스폰 위치가 `Vector3(randomX, spawnY, 0)` 기준을 유지하는지 확인한다.
9. Play Mode에서 설정한 간격마다 적이 생성되는지 확인한다.
10. 플레이어 사망 또는 게임 종료 상태에서 추가 스폰이 중지되는지 확인한다.
11. `Assets/Checklists/05_HO_EnemySpawner_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 8. `HO_UIController` 구현 및 UI 구성

1. `Assets/Tasks/06_HO_UIController.md` 기준으로 `Assets/Scripts/UI/HO_UIController.cs`를 구현한다.
2. 씬에 Canvas를 만든다.
3. Canvas는 Screen Space 기준으로 구성한다.
4. Canvas 기준 해상도를 세로형 `768 x 1024`에 맞춰 설정한다.
5. Canvas 하위에 HP 표시용 Slider를 만든다.
6. HP Slider를 화면 왼쪽 상단에 배치한다.
7. Canvas 하위에 점수 표시용 Text UI를 만든다.
8. 점수 Text를 화면 오른쪽 상단에 배치한다.
9. 씬에 빈 게임오브젝트를 만들고 이름을 `UIController`로 변경한다.
10. `UIController`에 `HO_UIController` 컴포넌트를 추가한다.
11. HP Slider 참조와 점수 Text 참조를 `HO_UIController`에 할당한다.
12. `GameManager`의 UI 컨트롤러 참조에 `UIController`를 할당한다.
13. 플레이어 HP가 변경될 때 HP Slider가 갱신되는지 확인한다.
14. 점수가 증가할 때 점수 Text가 갱신되는지 확인한다.
15. `768 x 1024` Game View에서 HP와 점수 UI가 상단에 정상 표시되는지 확인한다.
16. `Assets/Checklists/06_HO_UIController_Checklist.md`를 기준으로 완료 여부를 검증한다.

## 9. 전체 플레이 흐름 검증

1. Play Mode를 시작한다.
2. 플레이어가 XY 평면에서 화면 안을 자유롭게 이동하는지 확인한다.
3. `Fire1` 입력 시 총알이 `FirePoint`에서 생성되는지 확인한다.
4. 총알이 위쪽으로 이동하고 제거 Y 값을 넘으면 제거되는지 확인한다.
5. 적이 화면 위쪽에서 생성되어 아래쪽으로 이동하는지 확인한다.
6. 총알이 적에게 닿으면 총알과 적이 제거되는지 확인한다.
7. 총알로 적을 제거하면 점수가 증가하고 UI에 반영되는지 확인한다.
8. 적이 플레이어에게 닿으면 플레이어 HP가 감소하고 적이 제거되는지 확인한다.
9. 플레이어 HP가 0 이하가 되면 이동과 공격이 중지되는지 확인한다.
10. 플레이어 사망 이후 게임 종료 상태가 되고 적 스폰이 중지되는지 확인한다.
11. Unity Console에 컴파일 오류와 런타임 예외가 없는지 확인한다.
12. Game View `768 x 1024` 기준에서 카메라, 플레이어, 적, 총알, UI가 의도한 위치에 보이는지 확인한다.

## 10. 완료 보고 항목

1. 변경한 파일 경로를 적는다.
2. 구현한 Task 번호와 클래스명을 적는다.
3. Unity Console 컴파일 오류 여부를 적는다.
4. Play Mode에서 검증한 항목을 적는다.
5. 검증하지 못한 항목은 이유와 함께 적는다.
6. XY 평면, Orthographic 카메라, Old Input Manager 전제를 벗어난 변경이 있었다면 별도로 사유를 적는다.
