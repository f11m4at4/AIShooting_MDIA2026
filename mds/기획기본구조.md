# 게임 기획서: AIShooting

## [프로젝트명]

AIShooting

## 게임 개요

AIShooting은 `드래곤플라이트` 스타일의 2D Top View 비행 슈팅 게임이다. 플레이어는 세로형 화면에서 기체를 조작하며, 지속적으로 등장하는 적과 장애물을 피하고 공격해 높은 생존 시간과 점수를 목표로 한다.

기본 플레이는 XY 평면을 기준으로 진행되며, 세로형 기준 해상도 `768 x 1024`에 맞춘 직관적인 조작과 빠른 전투 템포를 지향한다.

## 핵심 컨셉

### 한 줄 컨셉

- 세로형 화면에서 기체를 조작해 몰려오는 적을 피하고 격추하며, 짧은 플레이 안에서 성장과 고득점 도전을 반복하는 탑뷰 로그라이크 슈팅 게임.

### 플레이 감각

- 조작은 단순하게 유지하고, 회피와 위치 선정의 비중을 높인다.
- 플레이어는 화면 전체를 자유롭게 움직이며 위쪽에서 내려오는 위협에 대응한다.
- 적은 빠르게 읽히는 패턴으로 등장해 즉각적인 판단을 요구한다.
- 한 판의 템포는 빠르게 시작하고, 시간이 지날수록 밀도와 난이도가 상승한다.

### 핵심 재미

- 제한된 공간 안에서 적을 아슬아슬하게 피하는 긴장감.
- 발사버튼을 누를때만 발사하는 구조로 적을 연속 격추하는 시원함.
- 플레이 중 적을 잡을때마다 획득한 점수로 점수가 올라가는 성취감.
- 짧은 실패 후 바로 재도전하며 이전 기록을 넘기는 반복 동기.

### 게임 정체성

- `드래곤플라이트` 스타일의 세로 진행 슈팅을 기반으로 한다.
- 2D Top View 감각을 유지하되 Unity 3D 오브젝트를 활용할 수 있다.
- 모든 주요 게임플레이 이동은 XY 평면에서 처리한다.
- 카메라는 Orthographic 기준으로 고정하고, 세로형 `768 x 1024` 화면 구성을 우선한다.

### 플레이어 목표

- 가능한 오래 생존한다.
- 적을 처치해 점수를 획득한다.
- 플레이 중 적을 공격해 획득한 점수를 높인다.
- 누적된 위협 속에서 최고 점수와 최고 생존 시간을 갱신한다.

### 핵심 구성 요소

- 플레이어 기체: 이동, 공격, 피격, 강화의 중심이 되는 주체.
- 적: 기본 적을 만든다.
- 투사체: 플레이어의 공격 수단이며, 명확한 속도와 충돌 판정을 가진다.
- 점수/시간: 플레이 성과를 측정하고 난이도 상승의 기준으로 사용한다.

### 설계 방향

- 첫 구현은 이동, 발사, 적 스폰, 충돌, 점수처럼 핵심 루프를 우선한다.
- 각 수치는 Inspector에서 조정 가능하게 만들어 빠른 밸런싱을 지원한다.
- 시스템은 작고 읽기 쉬운 단위로 나누되, 불필요한 추상화는 피한다.
- 화면 안의 위험 요소는 색, 크기, 이동 방향으로 즉시 구분되게 만든다.

## 핵심 메커니즘

### 목적

- 첫 프로토타입에서 플레이어 조작, 발사, 적 이동, 충돌 피드백이 정상적으로 연결되는지 검증한다.
- 모든 이동과 충돌은 XY 평면 기준으로 구성한다.
- 기능은 완성형 밸런스가 아니라 동작 검증이 가능한 최소 단위로 구현한다.

### 공통 전제

- 화면 기준: 세로형 `768 x 1024`.
- 카메라: Orthographic 고정.
- 입력: Old Input Manager 기반 `UnityEngine.Input` 사용.
- 좌표 기준: 플레이어, 적, 총알은 `Vector3(x, y, 0)` 위치에서 동작한다.
- 충돌 기준: 프로토타입에서는 단순 Collider 충돌로 판정한다.
- 주요 수치: 이동 속도, 발사 간격, HP, 대미지, 충돌 범위는 Inspector에서 조정 가능하게 둔다.

### 플레이어

#### 이동

- 플레이어는 화면 안에서 상하좌우로 이동한다.
- 입력은 `Horizontal`, `Vertical` 축을 사용한다.
- 대각선 이동 시 속도가 과하게 빨라지지 않도록 입력 벡터를 정규화한다.
- 화면 밖으로 나가지 않도록 이동 가능 영역을 제한한다.
- 프로토타입 기준으로 가속, 관성, 회전 연출은 적용하지 않는다.

#### 공격

- 플레이어는 공격 버튼을 눌렀을 때 한번 위쪽으로 총알을 발사한다.
- 발사 방향은 항상 월드 좌표 기준 `Vector3.up`이다.
- 발사 위치는 플레이어 전방의 고정 발사 지점으로 둔다.
- 프로토타입 기준으로 총알 종류, 강화, 산탄, 유도탄은 제외한다.

#### 피격

- 플레이어가 적과 충돌하면 HP가 감소한다.
- HP가 0 이하가 되면 플레이어는 사망 상태가 된다.
- 사망 상태에서는 이동과 공격을 중지한다.
- 프로토타입 기준으로 무적 시간, 넉백, 피격 애니메이션은 선택 사항으로 둔다.

#### 구현 체크리스트

- [ ] `HO_PlayerController`에서 XY 이동 입력을 처리한다.
- [ ] 플레이어 이동 속도를 Inspector에서 조정할 수 있다.
- [ ] 플레이어 이동 범위를 Inspector에서 조정할 수 있다.
- [ ] 화면 밖 이동이 제한된다.
- [ ] 공격 버튼을 누르면 총알이 생성된다.
- [ ] 총알 생성 위치를 별도 Transform으로 지정할 수 있다.
- [ ] 플레이어 HP를 Inspector에서 조정할 수 있다.
- [ ] 적과 충돌하면 HP가 감소한다.
- [ ] HP가 0 이하가 되면 이동과 공격이 멈춘다.

### 플레이어 총알

#### 이동

- 총알은 생성된 뒤 위쪽으로 단순 이동한다.
- 이동 방향은 항상 `Vector3.up`이다.
- 총알은 독립적인 속도를 가진다.
- 화면 위쪽 경계를 벗어나면 제거된다.
- 프로토타입 기준으로 관통, 튕김, 폭발, 유도는 제외한다.

#### 충돌

- 총알이 적과 충돌하면 적에게 대미지를 준다.
- 기본 프로토타입에서는 적을 즉시 제거해도 된다.
- 충돌 후 총알은 제거된다.

#### 구현 체크리스트

- [ ] `HO_PlayerBullet`에서 위쪽 단순 이동을 처리한다.
- [ ] 총알 이동 속도를 Inspector에서 조정할 수 있다.
- [ ] 화면 밖으로 나간 총알은 자동 제거된다.
- [ ] 적과 충돌하면 적 피격 처리를 호출한다.
- [ ] 충돌 후 총알이 제거된다.

### 적 (AI)

#### 이동

- 적은 화면 위쪽에서 생성되어 아래쪽으로 이동한다.
- 이동 방향은 기본적으로 `Vector3.down`이다.
- 프로토타입 기준으로 플레이어 추적, 회피, 복잡한 패턴 이동은 제외한다.
- 적이 화면 아래쪽 경계를 벗어나면 제거된다.

#### 공격

- 첫 프로토타입에서 적의 공격은 플레이어와의 접촉 공격으로 처리한다.
- 적이 플레이어와 충돌하면 플레이어에게 대미지를 준다.

#### 피격

- 적이 플레이어 총알에 맞으면 죽는다.
- 요구 검증 항목에 따라 적이 플레이어와 부딪히면 죽는다.
- 적 사망 시 오브젝트를 제거한다.
- 프로토타입 기준으로 사망 애니메이션, 드롭 아이템, 폭발 이펙트는 제외한다.

#### 구현 체크리스트

- [ ] `HO_Enemy`에서 아래쪽 단순 이동을 처리한다.
- [ ] 적 이동 속도를 Inspector에서 조정할 수 있다.
- [ ] 화면 아래쪽을 벗어난 적은 자동 제거된다.
- [ ] 플레이어와 충돌하면 플레이어 HP를 감소시킨다.
- [ ] 플레이어와 충돌한 적은 제거된다.
- [ ] 플레이어 총알에 맞은 적은 제거된다.

### UI
 - 플레이어 HP를 왼쪽 상단에 표시한다.
 - 플레이어 HP는 slider ui 이용해서 제작한다.
 - 플레이어가 적을 잡은 접수를 오른쪽 상단에 표시한다.

### 프로토타입 검증 흐름

1. 플레이어가 XY 평면에서 화면 안을 자유롭게 이동한다.
2. 공격 버튼을 누르면 총알이 위쪽으로 발사된다.
3. 적이 위쪽에서 생성되어 아래쪽으로 내려온다.
4. 총알이 적에게 닿으면 적이 제거된다.
5. 적이 플레이어에게 닿으면 플레이어 HP가 감소하고 적이 제거된다.
6. 플레이어 HP가 0이 되면 조작과 공격이 중지된다.
7. UI 가 정상적으로 화면에 잘 표시된다.

### 제외 항목

- 성장 시스템.
- 점수 보상 세부 규칙.
- 웨이브 난이도 곡선.
- 적 탄막 패턴.
- 보스전.
- 아이템 드롭.
- 사운드와 이펙트 세부 연출.

## 기술 명세

### 개발 전제

- Unity 버전은 `ProjectSettings/ProjectVersion.txt`의 `6000.4.10f1` 기준으로 작업한다.
- 주요 게임플레이는 XY 평면에서 동작하며, 플레이어, 적, 총알의 위치는 `Vector3(x, y, 0)`을 기준으로 사용한다.
- 주요 게임플레이 카메라는 Orthographic으로 구성하고 세로형 `768 x 1024` 화면 비율을 우선한다.
- 입력은 Old Input Manager 기반 `UnityEngine.Input` API를 사용한다.
- 새 런타임 클래스명은 `HO_` prefix를 붙인다.
- Inspector에 노출할 수치는 `[SerializeField] private` 필드를 우선 사용한다.
- 첫 구현은 프로토타입 검증이 목적이므로 이동, 발사, 적 스폰, 충돌, HP, 점수, UI 표시만 포함한다.
- 성장 시스템, 웨이브 난이도 곡선, 적 탄막, 아이템 드롭, 사운드와 이펙트 세부 연출은 제외한다.

### 생성 클래스 구조

| 클래스명 | 부모 클래스 | 역할 |
| --- | --- | --- |
| `HO_PlayerController` | `MonoBehaviour` | 플레이어 XY 이동, 화면 경계 제한, 공격 입력, HP 상태, 사망 상태를 관리한다. |
| `HO_PlayerBullet` | `MonoBehaviour` | 플레이어 총알의 위쪽 이동, 화면 밖 자동 제거, 적 충돌 시 대미지 전달을 처리한다. |
| `HO_Enemy` | `MonoBehaviour` | 적의 아래쪽 이동, 화면 밖 자동 제거, 플레이어 충돌 공격, 피격 사망 처리를 담당한다. |
| `HO_EnemySpawner` | `MonoBehaviour` | 화면 위쪽 스폰 영역에서 적 프리팹을 일정 간격으로 생성한다. |
| `HO_GameManager` | `MonoBehaviour` | 게임 진행 상태, 점수, 플레이어 사망 후 입력 중지 상태를 관리한다. |
| `HO_UIController` | `MonoBehaviour` | HP Slider와 점수 Text UI를 갱신한다. |

- 각 클래스는 단일 책임을 우선하며, 프로토타입 단계에서 범용 상속 구조나 복잡한 이벤트 버스는 만들지 않는다.
- 클래스 간 참조는 Inspector 할당을 우선하고, 런타임 검색은 초기화 시점에 필요한 최소 범위로 제한한다.
- 반복 사용하는 컴포넌트 참조는 `Awake` 또는 `Start`에서 캐싱한다.

### 플레이어 시스템

- `HO_PlayerController`에서 `Input.GetAxisRaw("Horizontal")`, `Input.GetAxisRaw("Vertical")`로 이동 입력을 읽는다.
- 입력 벡터는 `Vector2`로 만들고, 크기가 1을 초과하면 정규화한다.
- 이동은 `Vector3(input.x, input.y, 0)` 방향으로 처리하며 Z 값은 변경하지 않는다.
- 이동 속도는 `MoveSpeed` 성격의 `[SerializeField] private float` 값으로 조정한다.
- 이동 가능 범위는 `MinBounds`, `MaxBounds` 성격의 `[SerializeField] private Vector2` 값으로 조정한다.
- 이동 후 위치는 `Mathf.Clamp`로 화면 경계 안에 제한한다.
- 공격 입력은 `Input.GetButtonDown("Fire1")` 또는 명시된 Old Input Manager 버튼을 사용한다.
- 공격 입력이 들어오면 `BulletPrefab`을 `FirePoint` 위치에 생성한다.
- `FirePoint`가 비어 있으면 플레이어 Transform 위치를 기준으로 발사하되, 개발 중 확인 가능한 경고를 남긴다.
- 플레이어 HP는 최대 HP와 현재 HP로 분리하고, 최대 HP는 Inspector에서 조정 가능하게 둔다.
- 적과 충돌하면 적이 전달한 대미지만큼 HP를 감소시킨다.
- HP가 0 이하가 되면 사망 상태로 전환하고 이동과 공격 입력을 무시한다.

### 플레이어 총알 시스템

- `HO_PlayerBullet`은 생성 후 매 프레임 `Vector3.up` 방향으로 이동한다.
- 이동 속도는 Inspector에서 조정 가능한 `Speed` 성격의 값으로 둔다.
- 총알 대미지는 Inspector에서 조정 가능한 `Damage` 성격의 값으로 둔다.
- 총알은 XY 평면에서만 이동하며 Z 값을 변경하지 않는다.
- 화면 위쪽 제거 기준은 Inspector에서 조정 가능한 `DestroyY` 성격의 값으로 둔다.
- 총알이 `HO_Enemy`와 충돌하면 적의 피격 처리 메서드를 호출하고 자신을 제거한다.
- 프로토타입 단계에서는 관통, 유도, 폭발, 다중 타격은 구현하지 않는다.

### 적 시스템

- `HO_Enemy`는 생성 후 매 프레임 `Vector3.down` 방향으로 이동한다.
- 이동 속도는 Inspector에서 조정 가능한 `MoveSpeed` 성격의 값으로 둔다.
- 적 HP 또는 즉시 사망 여부는 프로토타입 검증 목적에 맞게 단순하게 유지한다.
- 기본 구현은 총알에 맞으면 제거되는 구조로 둔다.
- 적이 플레이어와 충돌하면 플레이어에게 접촉 대미지를 전달하고 자신을 제거한다.
- 화면 아래쪽 제거 기준은 Inspector에서 조정 가능한 `DestroyY` 성격의 값으로 둔다.
- 프로토타입 단계에서는 추적, 회피, 복합 이동 패턴, 원거리 공격을 구현하지 않는다.

### 적 스폰 시스템

- `HO_EnemySpawner`는 `EnemyPrefab`을 일정 간격으로 생성한다.
- 스폰 간격은 Inspector에서 조정 가능한 `SpawnInterval` 성격의 값으로 둔다.
- 스폰 위치는 화면 위쪽의 Y 값과 X 범위로 결정한다.
- X 스폰 범위는 Inspector에서 조정 가능한 `SpawnXMin`, `SpawnXMax` 성격의 값으로 둔다.
- Y 스폰 위치는 Inspector에서 조정 가능한 `SpawnY` 성격의 값으로 둔다.
- 게임이 종료되거나 플레이어가 사망한 상태에서는 추가 스폰을 중지한다.
- 첫 구현은 시간 기반 반복 스폰만 사용하고 웨이브 테이블이나 난이도 곡선은 제외한다.

### 충돌 및 대미지 처리

- 충돌 판정은 프로토타입 기준으로 Collider 충돌을 사용한다.
- 플레이어, 적, 총알은 충돌이 발생할 수 있도록 Collider를 가진다.
- Rigidbody 사용이 필요하면 XY 이동을 유지하고 Z축 이동이 발생하지 않도록 설정한다.
- 플레이어 피격 처리는 `HO_PlayerController`의 공개 메서드로 제공한다.
- 적 피격 처리는 `HO_Enemy`의 공개 메서드로 제공한다.
- 총알과 적 충돌 시 총알은 적에게 대미지를 전달한 뒤 제거된다.
- 적과 플레이어 충돌 시 플레이어 HP를 감소시키고 적은 제거된다.
- 사망 상태의 플레이어는 추가 입력을 받지 않으며, 중복 사망 처리가 발생하지 않도록 상태 값을 둔다.

### 점수 시스템

- 점수는 `HO_GameManager`에서 정수 값으로 관리한다.
- 플레이어 총알로 적을 제거했을 때 점수를 증가시킨다.
- 적 1마리당 획득 점수는 Inspector에서 조정 가능한 값으로 둔다.
- 플레이어와 충돌해 제거된 적은 기본적으로 처치 점수를 주지 않는다.
- 점수가 변경되면 `HO_UIController`에 갱신을 요청한다.
- 최고 점수 저장은 프로토타입 범위에서 제외한다.

### UI 시스템

- `HO_UIController`는 플레이어 HP Slider와 점수 Text를 참조한다.
- HP Slider는 화면 왼쪽 상단에 배치한다.
- 점수 Text는 화면 오른쪽 상단에 배치한다.
- UI Canvas는 세로형 `768 x 1024` 기준으로 구성한다.
- HP 값이 변경되면 Slider의 최대값과 현재값을 갱신한다.
- 점수 값이 변경되면 점수 Text를 갱신한다.
- UI는 게임플레이 로직을 직접 판단하지 않고 전달받은 값만 표시한다.

### 프리팹 구성

- 플레이어 프리팹에는 `HO_PlayerController`, Collider, 필요 시 Rigidbody를 포함한다.
- 플레이어 하위에는 총알 생성 위치를 나타내는 `FirePoint` Transform을 둔다.
- 플레이어 총알 프리팹에는 `HO_PlayerBullet`, Collider, 필요 시 Rigidbody를 포함한다.
- 적 프리팹에는 `HO_Enemy`, Collider, 필요 시 Rigidbody를 포함한다.
- 씬에는 `HO_EnemySpawner`, `HO_GameManager`, `HO_UIController`를 가진 오브젝트를 둔다.
- 프리팹의 시각 요소는 프로토타입 검증이 가능하도록 플레이어, 적, 총알이 서로 명확히 구분되게 한다.

### Inspector 조정 값

- 플레이어: 이동 속도, 이동 가능 최소 좌표, 이동 가능 최대 좌표, 최대 HP, 발사 간격이 필요한 경우의 쿨다운, 총알 프리팹, 발사 위치.
- 플레이어 총알: 이동 속도, 대미지, 화면 위쪽 제거 Y 값.
- 적: 이동 속도, 접촉 대미지, 화면 아래쪽 제거 Y 값, 처치 점수.
- 적 스포너: 적 프리팹, 스폰 간격, 스폰 X 최소값, 스폰 X 최대값, 스폰 Y 값.
- 게임 매니저: 적 처치 점수 기본값, 게임 시작 상태.
- UI: HP Slider 참조, 점수 Text 참조.

### 씬 구성

- 메인 카메라는 Orthographic Projection을 사용한다.
- 카메라는 XY 플레이 평면을 바라보며, 게임플레이 오브젝트는 `z = 0` 주변에서 동작한다.
- Game View 기준 해상도는 세로형 `768 x 1024`로 확인한다.
- 플레이어는 화면 하단 영역에서 시작하되, 전체 화면 안에서 이동 가능해야 한다.
- 적 스포너는 화면 위쪽 경계 밖 또는 경계 근처에서 적을 생성한다.
- Canvas는 Screen Space 기준으로 구성하고 HP와 점수가 지정된 상단 위치에 보이게 한다.

### 검증 항목

- 플레이어가 `Horizontal`, `Vertical` 입력으로 XY 평면에서 이동한다.
- 대각선 이동 시 이동 속도가 과하게 증가하지 않는다.
- 플레이어가 설정된 화면 경계 밖으로 나가지 않는다.
- 공격 버튼을 누르면 `FirePoint` 위치에서 총알이 생성된다.
- 총알이 월드 좌표 기준 위쪽으로 이동한다.
- 총알이 화면 위쪽 제거 기준을 넘으면 제거된다.
- 적이 화면 위쪽에서 생성되어 아래쪽으로 이동한다.
- 적이 화면 아래쪽 제거 기준을 넘으면 제거된다.
- 총알이 적과 충돌하면 적과 총알이 제거된다.
- 총알로 적을 제거하면 점수가 증가하고 UI에 반영된다.
- 적이 플레이어와 충돌하면 플레이어 HP가 감소하고 적이 제거된다.
- HP Slider가 현재 HP를 표시한다.
- 플레이어 HP가 0 이하가 되면 이동과 공격이 중지된다.
- 플레이어 사망 이후 적 스폰이 중지되거나 게임 종료 상태로 전환된다.
- Game View `768 x 1024` 기준에서 HP와 점수 UI가 정상적으로 보인다.

### 검증 원칙

- 검증은 프로토타입 핵심 루프가 실제 Play Mode에서 연결되는지 확인하는 것을 우선한다.
- 모든 이동 검증은 XY 평면 기준으로 수행하며, XZ 평면 이동이 섞이지 않았는지 확인한다.
- 카메라와 UI 검증은 세로형 `768 x 1024` Game View 기준으로 수행한다.
- 입력 검증은 Old Input Manager 축과 버튼 기준으로 수행한다.
- 수치 검증은 Inspector에서 값을 바꿨을 때 즉시 동작에 반영되는지 확인한다.
- 충돌 검증은 총알과 적, 적과 플레이어의 두 흐름을 분리해 확인한다.
- 사망 상태 검증은 HP가 0 이하가 된 뒤 이동, 공격, 스폰, UI가 일관된 상태를 유지하는지 확인한다.
- 구현 완료 판단은 체크리스트 항목을 통과하고 Unity 콘솔에 컴파일 오류가 없을 때로 한다.

### 응답 원칙

- 구현 결과를 보고할 때는 변경한 파일 경로, 구현한 시스템, 검증 여부를 짧게 정리한다.
- 검증하지 못한 항목은 성공처럼 표현하지 않고, 실행하지 못한 이유를 명확히 남긴다.
- Unity 버전, XY 평면, Orthographic 카메라, Old Input Manager 전제에서 벗어난 변경이 필요하면 먼저 별도 사유를 적는다.
- 요청 범위와 무관한 리팩터링, 네이밍 변경, 씬 재구성은 보고 없이 포함하지 않는다.
- 다른 작업자가 만든 변경을 발견하면 되돌리지 않고, 현재 작업에 영향이 있는 경우에만 언급한다.
