# 04. HO_GameManager 구현 Task

## 목표

게임 진행 상태, 점수, 플레이어 사망 후 게임 종료 상태를 관리하는 `HO_GameManager`를 구현한다.

## 구현 범위

- 런타임 스크립트 위치: `Assets/Scripts/Core/HO_GameManager.cs`
- 클래스명: `HO_GameManager`
- 부모 클래스: `MonoBehaviour`

## 구현 내용

1. 현재 게임 진행 상태를 관리한다.
2. 현재 점수를 정수 값으로 관리한다.
3. `EnemyScore` 성격의 `[SerializeField] private int` 값으로 적 1마리 처치 점수를 조정할 수 있게 한다.
4. 총알로 적을 제거했을 때 호출할 수 있는 점수 증가 메서드를 제공한다.
5. 점수가 변경되면 `HO_UIController`에 점수 갱신을 요청할 수 있는 구조를 만든다.
6. 플레이어 사망 시 호출할 수 있는 게임 종료 메서드를 제공한다.
7. 게임 종료 상태에서는 스포너가 추가 스폰을 중지할 수 있도록 상태 조회 메서드나 프로퍼티를 제공한다.
8. 최고 점수 저장, 재시작 UI, 웨이브 난이도 곡선은 구현하지 않는다.

## Inspector 조정 값

- 적 처치 점수
- 시작 시 게임 진행 여부
- UI 컨트롤러 참조

## 완료 기준

- 점수를 정수 값으로 관리한다.
- 적 처치 점수를 Inspector에서 바꿀 수 있다.
- 총알로 적을 제거했을 때 점수가 증가한다.
- 점수 변경이 UI 갱신 흐름으로 전달된다.
- 플레이어 사망 이후 게임 종료 상태로 전환된다.
- 게임 종료 상태를 다른 시스템에서 확인할 수 있다.
