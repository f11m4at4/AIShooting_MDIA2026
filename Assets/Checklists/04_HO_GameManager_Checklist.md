# 04. HO_GameManager 검증 체크리스트

## 컴파일 확인

- [ ] `HO_GameManager` 클래스명이 파일명과 일치한다.
- [ ] 새 클래스명에 `HO_` prefix가 붙어 있다.
- [ ] Inspector 노출 값은 `[SerializeField] private` 필드를 우선 사용한다.
- [ ] Unity Console에 컴파일 오류가 없다.

## 점수 검증

- [ ] 현재 점수를 정수 값으로 관리한다.
- [ ] 적 처치 점수를 Inspector에서 조정할 수 있다.
- [ ] 총알로 적을 제거했을 때 점수 증가 메서드가 호출된다.
- [ ] 점수 증가 후 현재 점수가 올바르게 누적된다.
- [ ] 플레이어와 충돌해 제거된 적은 기본 처치 점수를 주지 않는다.

## UI 연동 검증

- [ ] UI 컨트롤러 참조를 Inspector에서 할당할 수 있다.
- [ ] 점수 변경 시 `HO_UIController` 점수 갱신 메서드가 호출된다.
- [ ] UI 컨트롤러가 없어도 컴파일 오류 없이 동작하고 필요한 경우 경고만 남긴다.

## 게임 상태 검증

- [ ] 시작 시 게임 진행 상태를 설정할 수 있다.
- [ ] 플레이어 사망 시 게임 종료 상태로 전환된다.
- [ ] 게임 종료 상태를 다른 시스템에서 조회할 수 있다.
- [ ] 게임 종료 이후 스포너가 추가 스폰을 중지할 수 있는 값이 제공된다.
- [ ] 최고 점수 저장, 재시작 UI, 웨이브 난이도 곡선이 포함되지 않았다.
