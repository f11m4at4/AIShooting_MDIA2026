# 01. HO_PlayerController 검증 체크리스트

## 컴파일 확인

- [x] `HO_PlayerController` 클래스명이 파일명과 일치한다.
- [x] 새 클래스명에 `HO_` prefix가 붙어 있다.
- [x] `UnityEngine.Input` 기반 Old Input Manager API만 사용한다.
- [x] Inspector 노출 값은 `[SerializeField] private` 필드를 우선 사용한다.
- [ ] Unity Console에 컴파일 오류가 없다.

## 이동 검증

- [x] `Horizontal` 입력으로 X축 이동을 한다.
- [x] `Vertical` 입력으로 Y축 이동을 한다.
- [x] 이동은 XY 평면에서만 처리하고 XZ 평면 이동은 없다.
- [x] 대각선 입력 시 입력 벡터가 정규화되어 속도가 과하게 증가하지 않는다.
- [ ] 이동 속도를 Inspector에서 변경하면 Play Mode에서 반영된다.
- [x] 최소 경계값보다 작은 위치로 이동하지 않는다.
- [x] 최대 경계값보다 큰 위치로 이동하지 않는다.
- [x] 이동 시 Z 위치가 게임플레이 평면 이동 값으로 변하지 않는다.

## 공격 검증

- [x] `Fire1` 버튼 입력 시 총알 프리팹이 생성된다.
- [x] `FirePoint`가 할당되어 있으면 해당 위치에서 총알을 생성한다.
- [x] `FirePoint`가 비어 있으면 플레이어 위치에서 총알을 생성하고 경고가 출력된다.
- [x] 사망 상태가 아니면 공격 입력을 처리한다.
- [x] 사망 상태에서는 공격 입력을 무시한다.

## HP 및 사망 검증

- [x] 최대 HP를 Inspector에서 조정할 수 있다.
- [x] 피격 메서드를 호출하면 현재 HP가 감소한다.
- [x] HP가 0 이하가 되면 사망 상태로 전환된다.
- [x] 사망 상태에서는 이동 입력을 무시한다.
- [x] 사망 처리가 중복 실행되지 않는다.
